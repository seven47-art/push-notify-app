scheduled_at ISO 'T' separator datetime comparison bug - reproduction & verification
BUG: ISO 'T' separator causes datetime comparison failure in SQLite

scheduled_at is stored as 2026-03-21T11:25:00 (with T).
SQLite datetime('now') returns 2026-03-21 14:00:00 (with space).

In ASCII: 'T' (0x54) > ' ' (0x20), so string comparison yields:
'2026-03-21T11:25:00' > '2026-03-21 14:00:00' = true (even though 11:25 < 14:00)

Impact:
1. UPDATE ... SET status='triggered' WHERE scheduled_at <= datetime('now') -- never matches past T-formatted dates
2. SELECT COUNT(*) WHERE scheduled_at > datetime('now') -- counts past alarms as future
3. Result: ghost pending alarms inflate count, blocking new alarm creation
FIX: Normalize with replace(substr(scheduled_at,1,19),'T',' ')

- Line 155: auto-update past pending to triggered
- Line 163: channel pending alarm count
- Line 175: 5-min conflict check
- Line 481: trigger endpoint due-alarm query
